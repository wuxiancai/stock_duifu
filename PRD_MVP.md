# A股短线量化辅助决策系统 MVP 开发文档

## 1. 项目定位

本系统不是完整量化研究平台，也不是自动交易系统。

本系统的核心目标是：

每天盘后自动运行，帮助交易者判断市场环境、筛选强势板块、筛选候选股票，并生成第二天可执行的交易计划。

系统只做辅助决策，不直接下单。

## 2. MVP 核心流程

每天盘后运行以下流程：

1. 判断市场环境
2. 排出强势板块
3. 筛出候选股票
4. 给出买入条件
5. 给出止损价、止盈价、仓位
6. 第二天跟踪是否触发
7. 收盘后记录结果

系统最终输出两个核心结果：

- 今日交易计划
- 交易复盘记录

## 3. 目标用户

### 3.1 核心用户

A 股短线交易者。

用户需要：

- 每天快速知道市场是否适合交易
- 快速看到当前最强板块
- 快速看到符合模式的候选股票
- 明确第二天如何买、买多少、错了怎么止损
- 收盘后能复盘策略是否有效

## 4. MVP 不做什么

第一版明确不做以下功能：

- 不做实盘自动交易
- 不做复杂多数据源自动切换
- 不做完整 Feature Store 因子仓库
- 不做 Strategy Registry 插件化策略平台
- 不做 AI 自动决策
- 不做 JWT 多用户权限系统
- 不做复杂 DDD 架构
- 不做 IC / RankIC / 因子漂移监控
- 不做复杂撮合系统
- 不追求机构级量化平台

第一版只追求一件事：

让交易者每天能稳定得到一份清晰、可执行、可复盘的交易计划。

## 5. 系统核心模块

## 5.1 数据模块

### 5.1.1 数据范围

MVP 阶段只需要以下数据：

| 数据类型 | 用途 |
|---|---|
| A 股日线行情 | 判断趋势、涨跌幅、成交量 |
| 指数行情 | 判断大盘环境 |
| 板块行情 | 判断强势板块 |
| 个股基础信息 | 过滤 ST、退市、停牌、新股 |
| 涨跌停数据 | 判断市场情绪 |
| 成交额 / 换手率 | 判断流动性和资金活跃度 |
| 资金流 | 判断资金流向 |

### 5.1.2 数据源

第一版只接入一个主要数据源TuShare。主要数据源出错时，系统会自动切换到下一个数据源。

建议顺序：

1. TuShare
2. Akshare
3. 东方财富接口
4. 腾讯行情接口

### 5.1.3 数据更新时间

每天收盘后运行。

建议运行时间：

22:00 - 23:00

如数据源延迟，可手动重新运行。

## 5.2 市场环境判断模块

### 5.2.1 判断目标

判断明天是否适合交易。

市场状态分为四类：

| 状态 | 含义 | 操作建议 |
|---|---|---|
| 强势 | 市场赚钱效应好 | 可以正常交易 |
| 中性 | 市场震荡 | 轻仓参与 |
| 弱势 | 市场亏钱效应明显 | 降低仓位 |
| 风险 | 市场极弱或情绪退潮 | 空仓或只观察 |

### 5.2.2 核心指标

| 指标 | 说明 |
|---|---|
| 上证指数是否站上 20 日均线 | 判断大盘趋势 |
| 创业板指是否站上 20 日均线 | 判断成长股环境 |
| 全市场上涨家数 | 判断赚钱效应 |
| 全市场下跌家数 | 判断亏钱效应 |
| 涨停家数 | 判断短线情绪 |
| 跌停家数 | 判断风险 |
| 全市场成交额 | 判断资金活跃度 |
| 连板高度 | 判断短线接力情绪 |

### 5.2.3 市场评分规则

市场评分满分 100 分。

| 条件 | 分数 |
|---|---|
| 上证指数收盘价 > MA20 | +15 |
| 创业板指收盘价 > MA20 | +15 |
| 上涨家数 > 下跌家数 | +15 |
| 涨停家数 >= 40 | +15 |
| 跌停家数 <= 10 | +15 |
| 全市场成交额较昨日放大 | +10 |
| 连板高度 >= 3 板 | +15 |

### 5.2.4 市场状态划分

| 分数 | 市场状态 | 建议仓位 |
|---|---|---|
| >= 75 | 强势 | 80% - 100% |
| 55 - 74 | 中性 | 50% - 80% |
| 35 - 54 | 弱势 | 10% - 30% |
| < 35 | 风险 | 0% - 10% |

## 5.3 强势板块排序模块

### 5.3.1 判断目标

找出当前最强的板块。

系统每天输出 Top 10 强势板块。

### 5.3.2 板块评分指标

| 指标 | 说明 |
|---|---|
| 板块当日涨幅 | 判断板块强弱 |
| 板块 5 日涨幅 | 判断持续性 |
| 板块成交额变化 | 判断资金是否进入 |
| 板块内涨停数量 | 判断短线爆发力 |
| 板块内创新高股票数量 | 判断趋势强度 |
| 板块内强势股数量 | 判断板块扩散程度 |

### 5.3.3 板块评分规则

板块评分满分 100 分。

| 条件 | 分数 |
|---|---|
| 板块当日涨幅排名前 10% | +20 |
| 板块 3 日涨幅排名前 20% | +20 |
| 板块成交额较 5 日均值放大 | +20 |
| 板块内涨停数量 >= 2 | +20 |
| 板块内创新高股票数量 >= 3 | +10 |
| 板块内强势股数量 >= 5 | +10 |

### 5.3.4 输出结果

| 字段 | 说明 |
|---|---|
| 板块名称 | 如机器人、半导体、AI 应用 |
| 今日涨幅 | 板块当日涨幅 |
| 3 日涨幅 | 板块短期强度 |
| 成交额变化 | 是否放量 |
| 涨停数量 | 板块情绪 |
| 强势股数量 | 板块扩散 |
| 板块评分 | 0 - 100 |
| 排名 | Top 1 - Top 10 |

## 5.4 候选股票筛选模块

### 5.4.1 策略范围

MVP 阶段只保留 3 个策略：

1. 趋势强势策略
2. 放量突破策略
3. 强势回踩策略

不要同时开发太多策略。

### 5.4.2 股票基础过滤

先过滤掉不适合短线交易的股票。

| 条件 | 处理 |
|---|---|
| ST 股票 | 剔除 |
| 退市风险股票 | 剔除 |
| 停牌股票 | 剔除 |
| 上市不足 60 个交易日 | 剔除 |
| 当日成交额 < 5 亿 | 剔除 |
| 股价低于 2 元 | 剔除 |
| 一字涨停无法买入 | 标记观察，不作为买入计划 |

### 5.4.3 策略一：趋势强势策略

#### 策略逻辑

寻找已经进入上升趋势，并且趋势仍然健康的股票。但是要避免过度追高。龙头板块龙头股票，会强者恒强，可以适当放宽追高的条件。

#### 入选条件

| 条件 | 说明 |
|---|---|
| 收盘价 > MA5 > MA10 > MA20 | 均线多头排列 |
| 收盘价距离 MA20 不超过 25% | 避免过度追高 |
| 最近 20 日涨幅 > 15% | 有趋势强度 |
| 最近 5 日涨幅 > 3% | 短期仍然强势 |
| 成交额 >= 10 亿 | 保证流动性 |
| 所属板块进入 Top 10 强势板块 | 板块共振 |

#### 买入条件

第二天满足以下条件之一才允许买入：

| 买入方式 | 条件 |
|---|---|
| 回踩买入 | 盘中回踩 MA5 或 MA10 附近不破 |
| 突破买入 | 放量突破前一日高点 |
| 强势买入 | 开盘后站上 VWAP 且板块继续走强 |

### 5.4.4 策略二：放量突破策略

#### 策略逻辑

寻找放量突破平台或近期新高的股票。

#### 入选条件

| 条件 | 说明 |
|---|---|
| 收盘价突破最近 20 日新高 | 趋势突破 |
| 当日成交量 > 过去 5 日均量的 1.5 倍 | 放量确认 |
| 当日涨幅 3% - 8% | 避免过弱或过度高潮 |
| 收盘价 > MA5 > MA10 | 短期趋势健康 |
| 所属板块排名 Top 10 | 板块共振 |
| 当日不是一字板 | 第二天仍有参与可能 |

#### 买入条件

第二天满足以下条件才允许买入：

| 条件 | 说明 |
|---|---|
| 股价不低开超过 3% | 避免弱转强失败 |
| 盘中突破昨日高点 | 确认继续走强 |
| 成交量同步放大 | 避免假突破 |
| 板块排名仍在前列 | 避免个股孤立上涨 |

### 5.4.5 策略三：强势回踩策略

#### 策略逻辑

寻找前期强势上涨后，短线回踩但趋势未破坏的股票。

#### 入选条件

| 条件 | 说明 |
|---|---|
| 最近 20 日涨幅 > 20% | 前期强势 |
| 当前收盘价仍在 MA20 上方 | 趋势未破 |
| 回踩 MA10 或 MA20 附近 | 等待低吸机会 |
| 回调期间成交量缩小 | 抛压减弱 |
| 所属板块仍然强势 | 板块未退潮 |
| 跌幅没有破坏结构 | 避免趋势转弱 |

#### 买入条件

第二天满足以下条件才允许买入：

| 条件 | 说明 |
|---|---|
| 盘中不跌破 MA20 | 趋势保护 |
| 出现放量反包或站回 MA5 | 回踩结束信号 |
| 板块没有明显退潮 | 避免逆势低吸 |
| 大盘状态不是风险 | 避免弱市接飞刀 |

## 5.6 止损、止盈、仓位规则

### 5.6.1 止损规则

止损价取以下三者中更保守的一个：

| 类型 | 规则 |
|---|---|
| 技术止损 | 跌破 MA10 或 MA20 |
| 固定止损 | 买入价下跌 5% |
| ATR 止损 | 买入价 - 1.5 × ATR14 |

最终止损价选择：

止损价 = max(技术止损价, 固定止损价, ATR止损价)

说明：

这里的 max 表示距离买入价更近的止损，优先控制亏损。

### 5.6.2 止盈规则

MVP 使用简单止盈规则：

| 条件 | 操作 |
|---|---|
| 盈利达到 20% | 卖出 50% |
| 跌破 MA5 | 卖出剩余仓位 |
| 涨停未开板 | 可继续持有 |
| 放量长上影 | 次日优先减仓 |

### 5.6.3 仓位规则

根据市场状态决定总仓位。

| 市场状态 | 总仓位上限 | 单票仓位 |
|---|---|---|
| 强势 | 100% | 50% |
| 中性 | 80% | 20% - 40% |
| 弱势 | 30% | 5% - 10% |
| 风险 | 10% | 只观察或极小仓 |

单日最多新开仓：

| 市场状态 | 最多新开 |
|---|---|
| 强势 | 3 只，且不能都是同一个板块 |
| 中性 | 2 只 |
| 弱势 | 1 只 |
| 风险 | 0 只 |

## 5.7 第二天盘中跟踪模块

### 5.7.1 跟踪目标

第二天盘中跟踪交易计划是否触发。

系统不自动下单，只标记状态。

### 5.7.2 状态定义

| 状态 | 含义 |
|---|---|
| 待触发 | 计划已生成，尚未满足买入条件 |
| 已触发 | 满足买入条件 |
| 未触发 | 全天未满足买入条件 |
| 取消 | 市场或板块走弱，不再执行 |
| 风险 | 触发后快速走弱，需要谨慎 |

### 5.7.3 取消条件

出现以下情况时，取消交易计划：

| 条件 | 说明 |
|---|---|
| 大盘状态变为风险 | 系统性风险 |
| 所属板块跌幅居前 | 板块退潮 |
| 个股低开超过 5% | 预期明显转弱 |
| 个股跌破关键均线 | 结构破坏 |
| 跌停或接近跌停 | 流动性风险 |

### 5.8.1 复盘目标

### 5.8.2 复盘字段

| 收盘价 | 当日收盘价 |
| 次日收益 | T+1 收益 |
| 5日收益 | T+5 收益 |
| 最大浮盈 | 计划周期内最高收益 |
| 最大浮亏 | 计划周期内最大亏损 |
| 是否止损 | 是 / 否 |
| 是否止盈 | 是 / 否 |

## 15. 关键设计原则

### 15.1 先可用，再复杂

第一版不追求完美，只追求每天能用。

### 15.2 先验证策略，再优化架构

如果策略不能赚钱，架构再漂亮也没意义。

### 15.3 所有信号必须能解释

每只股票为什么入选，必须有清楚理由。

### 15.4 所有计划必须有风控

没有止损价的股票，不允许进入交易计划。

### 15.5 系统只给计划，不替人下单

交易者最终决定是否执行。

## 16. 最终目标

MVP 的最终目标不是做一个庞大的量化平台，而是做一个简单、稳定、每天能用的短线交易辅助系统。

最终用户每天只需要看三件事：

1. 今天市场能不能做
2. 哪些板块最强
3. 明天哪些股票值得重点关注

系统核心价值：

减少主观冲动，提高交易纪律，让每一笔交易都有计划、有风控、有复盘。

## 17. 扩展模块

## 17.1 模拟交易模块

### 17.1.1 模块定位

模拟交易模块用于验证交易计划在接近真实交易环境下的表现。

该模块不接入券商 API，不使用真实资金，但执行逻辑应尽量接近真实交易。

模拟交易模块的核心目标是：

1. 验证交易计划是否具有实战可执行性
2. 验证买入条件触发后的真实收益表现
3. 验证止损、止盈、仓位规则是否有效
4. 统计资金曲线、胜率、盈亏比、最大回撤
5. 为未来是否接入真实交易接口提供依据

### 17.1.2 与交易计划的关系

模拟交易模块必须基于系统生成的交易计划运行。

当天收盘后，系统生成第二天交易计划。

第二天交易时，模拟交易模块通过实时或延迟行情数据跟踪交易计划。

如果买入条件触发，模拟交易模块按照规则自动模拟买入。

如果止损、止盈或卖出条件触发，模拟交易模块按照规则自动模拟卖出。

模拟交易只执行系统计划内的交易，不允许执行计划外交易。

### 17.1.3 初始资金

模拟交易默认初始资金为：

100 万元

资金使用复利模式。

如果今日盈利，盈利金额自动计入下一交易日可用资金。

如果今日亏损，亏损金额自动从下一交易日可用资金中扣除。

示例：

| 日期 | 期初资金 | 当日盈亏 | 期末资金 | 次日期初资金 |
|---|---|---|---|---|
| 第 1 天 | 1,000,000 | +200,000 | 1,200,000 | 1,200,000 |
| 第 2 天 | 1,200,000 | -60,000 | 1,140,000 | 1,140,000 |

### 17.1.4 资金账户字段

模拟账户需要记录以下字段：

| 字段 | 说明 |
|---|---|
| account_id | 模拟账户 ID |
| account_name | 模拟账户名称 |
| initial_cash | 初始资金 |
| total_assets | 总资产 |
| available_cash | 可用现金 |
| market_value | 持仓市值 |
| frozen_cash | 冻结资金 |
| total_profit | 累计盈亏 |
| total_return | 累计收益率 |
| max_drawdown | 最大回撤 |
| created_at | 创建时间 |
| updated_at | 更新时间 |

### 17.1.5 交易费用规则

模拟交易必须计入交易成本。

默认费用参数如下：

| 费用类型 | 默认规则 | 收取方向 |
|---|---|---|
| 佣金 | 成交金额 × 0.03% | 买入和卖出双边 |
| 印花税 | 成交金额 × 0.05% | 仅卖出 |
| 过户费 | 成交金额 × 0.001% | 买入和卖出双边 |
| 最低佣金 | 5 元 | 单笔交易 |

说明：

1. 佣金默认使用 0.03%，即万分之三。
2. 印花税默认使用 0.05%，即万分之五。
3. 过户费默认使用 0.001%，即十万分之一。
4. 所有费用参数必须支持配置。
5. 后续如果费率变化，只需要修改配置，不修改交易逻辑。

### 17.1.6 买入规则

模拟交易模块只允许买入交易计划中的股票。

买入触发条件来自 trade_plan 表中的买入条件。

当实时行情满足买入条件时，模拟交易模块执行模拟买入。

买入价格规则：

| 情况 | 模拟成交价 |
|---|---|
| 突破买入 | 触发价或当前价 |
| 回踩买入 | 买入区间内价格 |
| 强势买入 | 站上 VWAP 后当前价 |
| 涨停无法买入 | 不成交，标记为未成交 |
| 高开超过限制 | 取消买入 |

买入数量规则：

1. 根据建议仓位计算计划买入金额。
2. 买入金额不能超过可用现金。
3. 买入股数必须按 100 股整数倍计算。
4. 扣除佣金、过户费后现金不足时，自动减少买入股数。
5. 如果可买数量小于 100 股，则不买入。

买入金额计算：

计划买入金额 = 当前总资产 × 建议仓位

实际买入金额 = 成交价格 × 买入股数

买入总成本 = 实际买入金额 + 买入佣金 + 买入过户费

### 17.1.7 卖出规则

模拟交易模块根据交易计划中的止损、止盈、移动止盈规则自动卖出。

卖出触发条件包括：

| 条件 | 操作 |
|---|---|
| 跌破止损价 | 全部卖出 |
| 盈利达到第一止盈位 | 卖出 50% |
| 盈利达到第二止盈位 | 再卖出 30% |
| 跌破 MA5 | 卖出剩余仓位 |
| 市场状态变为风险 | 降低仓位或全部卖出 |
| 板块明显退潮 | 降低仓位或全部卖出 |
| 持仓超过计划周期 | 按收盘价卖出或标记待处理 |

卖出价格规则：

| 情况 | 模拟成交价 |
|---|---|
| 正常卖出 | 当前价 |
| 止损卖出 | 止损触发价或当前价 |
| 止盈卖出 | 止盈触发价或当前价 |
| 跌停无法卖出 | 不成交，标记为跌停未成交 |
| 快速跳水 | 当前价，并记录滑点 |

卖出费用包括：

1. 卖出佣金
2. 印花税
3. 卖出过户费

卖出到账金额：

卖出到账金额 = 成交金额 - 卖出佣金 - 印花税 - 卖出过户费

### 17.1.16 模拟交易定时任务

模拟交易模块需要在交易时段运行。

运行时间：

交易日 09:30 - 15:00

运行频率：

每 1 - 5 分钟运行一次

执行流程：

1. 读取当前有效交易计划
2. 读取当前持仓
3. 获取实时或延迟行情
4. 判断计划是否触发买入
5. 判断持仓是否触发卖出
6. 生成模拟交易记录
7. 更新持仓
8. 更新账户资产
9. 更新资金曲线

### 17.1.17 模拟交易成功标准

模拟交易模块完成后，应满足以下标准：

1. 默认使用 100 万初始资金
2. 支持复利模式
3. 能根据交易计划自动模拟买入
4. 能根据止损止盈规则自动模拟卖出
5. 能计算佣金、印花税、过户费
6. 能展示当前持仓
7. 能展示当天交易记录
8. 每笔买卖都有原因
9. 能展示总资产、可用现金、持仓市值
10. 能统计累计收益率和最大回撤

### 17.1.18 模拟交易设计原则

模拟交易不是为了美化收益，而是为了暴露真实问题。

必须尽量保守处理以下情况：

1. 涨停买不到
2. 跌停卖不出
3. 高开过多不追
4. 低开破位取消
5. 手续费必须扣除
6. 印花税必须扣除
7. 买入数量必须按 100 股整数倍
8. 现金不足不能买入
9. 计划外股票不能买入
10. 没有止损价的计划不能买入

### 17.1.19 与 MVP 的关系

模拟交易模块属于扩展模块，但建议优先级提高。

原因：

1. 它能验证策略是否真正可执行
2. 它能减少主观判断干扰
3. 它能统计真实资金曲线
4. 它能为未来接入实盘交易接口打基础

建议开发顺序：

1. 先完成交易计划生成
2. 再完成模拟买入
3. 再完成模拟卖出
4. 再完成持仓页面
5. 再完成交易记录页面
6. 最后完成资金曲线和统计指标
