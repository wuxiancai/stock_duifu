# Decisions

## DEC-0001: 本仓库是全新系统

- 状态：Accepted
- 日期：2026-06-20

本仓库从 `PRD_MVP.md` 开始重新建设。旧 `stock` 项目的实现、部署、测试、验收结论和历史假设全部作废，不作为当前系统事实来源。

## DEC-0002: 系统只做辅助决策，不做自动实盘交易

- 状态：Accepted
- 日期：2026-06-20

系统输出交易计划、跟踪状态和复盘结果，但不直接下单，不接管真实账户。交易者最终决定是否执行。

## DEC-0003: 真实数据优先

- 状态：Accepted
- 日期：2026-06-20

MVP 必须面向真实 A 股行情数据建设。mock 只能用于开发占位或测试边界，不能作为 MVP 完成依据。验收必须提供真实数据、真实数据库、真实 API 或真实页面证据。

## DEC-0004: PostgreSQL 作为长期默认数据库

- 状态：Accepted
- 日期：2026-06-20

MVP 文档允许 SQLite 或 PostgreSQL，但本项目长期默认选择 PostgreSQL。SQLite 只能作为临时本地试验，不作为部署和验收默认方案。

## DEC-0005: 先完成 P0 闭环，再扩展平台能力

- 状态：Accepted
- 日期：2026-06-20

开发顺序优先完成数据采集、市场环境、强势板块、候选筛选、交易计划、P0 Web 页面和复盘。完整 Feature Store、Strategy Registry、多用户权限和 AI 自动决策都不进入第一版。

## DEC-0006: 默认前端采用 Vue 3 + Element Plus

- 状态：Accepted
- 日期：2026-06-20

本系统以表格、筛选、状态面板和复盘为主，默认使用 Vue 3 + Element Plus 以提高后台工具开发效率。除非后续明确变更，不采用 Next.js 作为默认前端栈。

## DEC-0007: 每个开发节点必须 git 留痕

- 状态：Accepted
- 日期：2026-06-20

每次开发必须先检查 git 状态，每个有意义节点提交一次。任务完成或中断前必须更新 `docs/HANDOFF.md` 并提交，保证后续 agent 能从仓库恢复上下文。

## DEC-0009: 拉取数据统一走多源切换系统

- 状态：Accepted
- 日期：2026-07-07

所有涉及外部数据拉取的入口必须逐步收敛到统一 `DataSourceRouter`。业务模块不得直接依赖单一源；首选源失败、超时、返回空数据或脏数据时，应按数据域策略切换备用源。所有源失败且无安全缓存或推导兜底时，不允许强行生成交易计划。

## DEC-0008: 股票池优先识别行业持续性，而非平均分配行业名额

- 状态：Accepted
- 日期：2026-06-26

股票池生成必须遵循“强者恒强”的 A 股短线原则：先识别行业趋势结构，再在行业内选股票。行业持续性判断使用极限 Top5 口径，不使用 Top10 作为股票池主逻辑门槛。单日异动行业不进入股票池主逻辑；连续多日 Top5 强势的行业应优先入选。行业配额不再按“每个行业先选 1 只”平均分配，而是按行业持续性动态分配：核心主升行业可多选，稳定强势行业至少保留 2 只，弱持续性或单日异动行业剔除或降权。个股评分需要叠加行业持续性加分和风险修正，今日交易计划再从股票池中进一步筛选生成。
