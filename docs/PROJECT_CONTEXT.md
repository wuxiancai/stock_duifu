# Project Context

## 项目定位

本项目是全新的 A 股短线量化辅助决策系统，从 `PRD_MVP.md` 开始建设，不继承旧 `stock` 项目的任何代码、部署方式、测试结论或业务假设。

系统不是完整量化研究平台，也不是自动交易系统。核心目标是每天盘后自动运行，帮助交易者判断市场环境、筛选强势板块、筛选候选股票，并生成第二天可执行的条件交易计划。

## 目标用户

核心用户是 A 股短线交易者。用户每天需要快速回答三件事：

1. 今天市场能不能做。
2. 哪些板块最强。
3. 明天哪些股票值得重点关注，以及错了怎么止损。

## MVP 核心流程

1. 盘后拉取真实行情数据。
2. 判断市场环境：强势、中性、弱势、风险。
3. 排出 Top 10 强势板块。
4. 用三类策略筛出候选股票：趋势强势、放量突破、强势回踩。
5. 生成第二天交易计划：买入条件、买入区间、止损价、止盈价、建议仓位。
6. 第二天盘中跟踪是否触发或取消。
7. 收盘后生成复盘记录和策略统计。

## P0 页面

第一版必须完成以下 P0 页面：

- 今日决策面板：市场状态、市场评分、涨跌停数量、上涨/下跌家数、成交额、系统建议。
- 强势板块页面：Top 10 板块排名、涨幅、成交额变化、涨停数量、强势股数量、评分。
- 今日交易计划页面：候选股票、策略、评分、买入条件、止损、止盈、仓位、状态和风险提示。
- 交易复盘页面：触发、收益、最大浮盈/浮亏、结果、失败原因、策略/板块统计。

## MVP 成功标准

- 每天盘后能自动生成市场状态。
- 每天能输出 Top 10 强势板块。
- 每天能输出候选股票交易计划。
- 每只候选股票都有明确买入条件、止损价、止盈价和建议仓位。
- Web 页面能清楚展示交易计划。
- 第二天能标记计划是否触发或取消。
- 收盘后能生成复盘记录。
- 最近 30 天能统计策略胜率和盈亏表现。
- 用户每天打开系统后，5 分钟内能完成次日交易准备。

## 技术默认

- 后端默认使用 Python + FastAPI。
- 前端默认使用 Vue 3 + Element Plus。
- 数据库默认使用 PostgreSQL；SQLite 只能作为临时本地试验，不作为长期默认方案。
- 行情数据真实数据优先，数据源优先级为 TuShare、AkShare、东方财富、腾讯行情。
- 任务调度可以先用简单定时任务实现，后续再根据运行稳定性选择更重的调度系统。

## 非目标

MVP 不做实盘自动交易、不做复杂多数据源自动切换、不做完整 Feature Store、不做 Strategy Registry 插件化平台、不做 AI 自动决策、不做多用户权限系统、不追求机构级量化平台。
